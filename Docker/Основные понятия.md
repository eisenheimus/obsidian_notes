
# Основные отличия виртуальных машин от контейнеров

| Аспект         | Виртуальные машины (VM)                                    | Контейнеры                                                      |
| -------------- | ---------------------------------------------------------- | --------------------------------------------------------------- |
| **Изоляция**   | Полная: каждая VM имеет своё ядро ОС, драйверы, библиотеки | Процессная: общее ядро хоста, изоляция через namespaces/cgroups |
| **Гипервизор** | Требуется (KVM, VMware, Hyper-V)                           | Не требуется, работает поверх ядра Linux                        |
| **Образ**      | Полный образ ОС (ГБ)                                       | Минимальный образ приложения (МБ)                               |


Виртуальные машины:
✅ Полная изоляция и безопасность
✅ Можно запускать разные ОС (Linux, Windows, BSD)
❌ Высокие накладные расходы (CPU/RAM)
❌ Долгий запуск (десятки секунд)
❌ Большой расход дискового пространства

Контейнеры:
✅ Быстрый запуск (миллисекунды)
✅ Минимальные накладные расходы
✅ Эффективное использование ресурсов
✅ Идеальны для микросервисов и CI/CD
❌ Общее ядро = меньше изоляции
❌ Только ОС хоста (Linux-контейнеры на Linux)

> **Гипервизор** — это специальная программа-«менеджер», которая позволяет запустить несколько независимых виртуальных компьютеров (виртуальных машин) на одном реальном физическом сервере. Он делит ресурсы (процессор, память, диски) между ними, обеспечивая их работу без влияния друг на друга.

> **Namespaces и cgroups** - фундамент контейнеризации в Linux
   Это две ключевые технологии ядра Linux, которые делают возможными контейнеры (Docker, LXC, Podman). Понимание этих механизмов критично для DevOps/SRE и входит в программу LPIC-1/2.

**Namespaces** изолируют глобальные ресурсы ядра, чтобы процесс внутри контейнера «видел» только своё окружение.

**Control Groups (cgroups)** ограничивают, учитывают и изолируют использование ресурсов (CPU, RAM, disk I/O, сеть) для группы процессов.
