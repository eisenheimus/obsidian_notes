#### <font color=yellow>**Детализация пакета транспортного уровня (L3):**</font>


**Версия (Version)**
Версия (4 бита) - указывает версию протокола IP (0100 = IPv4, 0110 = IPv6). Получатель отбрасывает пакет, если не поддерживает указанную версию.

**Длина заголовка (IHL - Internet Header Length)**
Длина заголовка (4 бита) - количество 32-битных слов в заголовке. Минимальное значение 5 (20 байт), максимальное 15 (60 байт с опциями). Нужно, чтобы понять, где заканчивается заголовок и начинаются данные.

**Тип обслуживания (Type of Service / DSCP+ECN)**
Тип обслуживания (8 бит) - приоритет пакета и параметры качества обслуживания. DSCP (6 бит) для классификации трафика (VoIP, видео, обычный), ECN (2 бита) для уведомления о перегрузке без потери пакетов.

**Общая длина (Total Length)**
Общая длина (16 бит) - полный размер пакета в байтах (заголовок + данные). Максимум 65 535 байт. Получатель использует это поле для определения конца пакета.

**Идентификатор (Identification)**
Идентификатор (16 бит) - уникальное значение для каждого пакета. Используется при фрагментации: все фрагменты одного исходного пакета имеют одинаковый ID, чтобы получатель мог их правильно собрать.

**Флаги (Flags)**
Флаги (3 бита) - управление фрагментацией:
• Бит 0: зарезервирован (всегда 0)
• Бит 1 (DF - Don't Fragment): запрет на фрагментацию (используется в Path MTU Discovery)
• Бит 2 (MF - More Fragments): 1 = есть ещё фрагменты, 0 = последний или единственный фрагмент

**Смещение фрагмента (Fragment Offset)**
Смещение фрагмента (13 бит) - указывает позицию данного фрагмента относительно начала исходного пакета (в единицах по 8 байт). Позволяет собрать пакет в правильном порядке даже если фрагменты пришли не по порядку.

**Время жизни (TTL - Time To Live)**
Время жизни (8 бит) - «счётчик прыжков». Уменьшается на 1 каждым маршрутизатором. При достижении 0 пакет отбрасывается, отправителю уходит ICMP Time Exceeded. Защита от зацикливания пакетов в сети.

**Протокол (Protocol)**
Протокол (8 бит) - указывает, какой протокол транспортного уровня «упакован» в данные: 1=ICMP, 6=TCP, 17=UDP, 47=GRE, 50=ESP (IPsec) и т.д. Нужен, чтобы передать данные правильному обработчику на принимающей стороне.

**Контрольная сумма заголовка (Header Checksum)**
Контрольная сумма (16 бит) - вычисляется только по заголовку для обнаружения ошибок. Пересчитывается каждым роутером (так как меняется TTL). Если чексумма не совпадает — пакет отбрасывается без уведомления.

**Исходящий IP-адрес (Source IP Address)**
Исходящий адрес (32 бита) - логический адрес отправителя в формате IPv4 (например, 192.168.1.10). Используется для маршрутизации ответа и фильтрации (ACL, фаерволы).

**Адрес назначения (Destination IP Address)**
Адрес назначения (32 бита) - логический адрес получателя (например, 8.8.8.8). Основной критерий для выбора пути в таблицах маршрутизации.

**Опции (Options) - необязательно**
Опции (переменная длина) - дополнительные функции: запись маршрута (Record Route), метка времени (Timestamp), строгая/нестрогая маршрутизация (LSR/SSR). Используется редко, может обрабатываться медленнее.

**Заполнение (Padding)**
Заполнение (переменная длина) - нулевые байты, чтобы длина заголовка была кратна 32 битам (требование стандарта).

#### <font color=yellow>**Отправка сообщения с одного ПК на другой ПК**</font>

Компьютер: 192.168.1.10/24
Маска: 255.255.255.0
Шлюз по умолчанию: 192.168.1.1

**Сценарий А: Отправка на 192.168.1.20 (в той же подсети)**
┌─────────────────────────────────────────┐
│ 1. Сравниваем: 192.168.1.20 & 255.255.255.0 = 192.168.1.0 │
│ 2. Наша сеть: 192.168.1.10 & 255.255.255.0 = 192.168.1.0 │
│ 3. Результат: СОВПАДАЕТ → адрес локальный               │
│ 4. Действие: Ищем MAC адресата через ARP                │
│ 5. Отправляем кадр напрямую: dst MAC = MAC получателя   │
└─────────────────────────────────────────┘

**Сценарий Б: Отправка на 8.8.8.8 (в другой сети)**
┌─────────────────────────────────────────┐
│ 1. Сравниваем: 8.8.8.8 & 255.255.255.0 = 8.8.8.0        │
│ 2. Наша сеть: 192.168.1.10 & 255.255.255.0 = 192.168.1.0│
│ 3. Результат: НЕ СОВПАДАЕТ → адрес внешний              │
│ 4. Действие: Ищем MAC шлюза (192.168.1.1) через ARP     │
│ 5. Отправляем кадр шлюзу: dst MAC = MAC роутера         │
│    (но dst IP в пакете остаётся 8.8.8.8!)               │
└─────────────────────────────────────────┘

**Маршрутизатор получил кадр (dst MAC = его собственный)**
┌─────────────────────────────────────────────────────┐
│ 1. Снимает L2-заголовок (декапсуляция)              │
│ 2. Смотрит на dst IP в пакете (например, 8.8.8.8)   │
│ 3. Ищет в ТАБЛИЦЕ МАРШРУТИЗАЦИИ лучший маршрут      │
│                                                     │
│ 4. Варианты:                                        │
│    • Есть прямая сеть → отправляет напрямую         │
│    • Есть следующий хоуп → отправляет дальше        │
│    • Нет маршрута → отбрасывает + ICMP unreachable  │
│                                                     │
│ 5. Для следующего хопа ищет MAC через ARP           │
│ 6. Инкапсулирует в НОВЫЙ L2-кадр (MAC меняются!)    │
│ 7. Отправляет дальше                                │
└─────────────────────────────────────────────────────┘
