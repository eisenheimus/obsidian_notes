
>Модель OSI (Open Systems Interconnection) — это эталонная модель взаимодействия открытых систем, описывающая, как данные передаются от одного устройства к другому по сети. Рассмотрим все 7 уровней **снизу вверх**, начиная с физического уровня (L1), с примерами, протоколами и тем, что именно происходит на каждом этапе.

## 7 уровней


| Уровень       | №   | Описание                                                                                                                                                                      | Единица передачи     | Пример                                                                                                                                                                                                                                                                                                                               | Протоколы                                                                                                                                                                                                                              |
| ------------- | --- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Прикладной    | L7  | Предоставляет сетевые сервисы непосредственно пользовательским приложениям. Это интерфейс между программой и сетью.                                                           | Сообщение            |                                                                                                                                                                                                                                                                                                                                      | HTTP/HTTPS <br>SSH<br>FTP/SFTP<br>DHCP<br>SNMP                                                                                                                                                                                         |
| Представления | L6  | Преобразует данные в формат, понятный приложению: шифрование, сжатие, кодирование, трансляция форматов.                                                                       | Данные представления | Веб-сервер отправляет страницу:<br><br>1. Данные в HTML сжимаются через gzip (L6)<br>2. Шифруются через TLS (формально между L4 и L7)<br>3. Передаются дальше<br><br>Браузер получает данные, расшифровывает, распаковывает и отображает.                                                                                            | - **Шифрование/дешифрование**: TLS/SSL (хотя часто относят к L4–L7)<br>- **Сжатие**: gzip, deflate в HTTP<br>- **Кодирование**: ASCII ↔ Unicode, Base64 для вложений в email<br>- **Трансляция форматов**: JPEG, PNG, MPEG, XML ↔ JSON |
| Сеансовый     | L5  | Управляет диалогом между приложениями: устанавливает, поддерживает, синхронизирует и завершает сеансы связи.                                                                  | Данные сеанса        | При загрузке большого файла по FTP:<br><br>- Устанавливается сеанс между клиентом и сервером<br>- Если соединение обрывается на 70%, сеансовый уровень может позволить возобновить передачу с места разрыва (если протокол поддерживает)<br>- Управление диалогом: кто говорит, когда, в каком порядке                               | - **RPC (Remote Procedure Call)**<br>- **NetBIOS**, **PPTP**<br>- **SQL sessions**, **SSH session management**<br>- Механизмы контрольных точек (checkpoints) для возобновления передачи                                               |
| Транспортный  | L4  | Обеспечивает сквозную (end-to-end) передачу данных между приложениями на разных хостах. Управляет потоком, надёжностью, мультиплексированием соединений.                      | Сегмент (TCP)        | Браузер открывает соединение с веб-сервером:<br><br>1. **Three-way handshake**: `SYN` → `SYN-ACK` → `ACK`<br>2. Передача данных с нумерацией сегментов, подтверждениями (ACK), повторной передачей при потере<br>3. Завершение: `FIN` / `ACK`                                                                                        | TCP/UDP                                                                                                                                                                                                                                |
| Сетевой       | L3  | Обеспечивает логическую адресацию и маршрутизацию между разными сетями. Отвечает за выбор пути доставки пакета от источника к получателю через множество промежуточных узлов. | Пакет (packet)       | Компьютер `192.168.1.10` отправляет данные на сервер `8.8.8.8`. Он понимает, что адрес не в локальной сети, и отправляет пакет на шлюз по умолчанию (`192.168.1.1`). Маршрутизатор смотрит в таблицу маршрутизации и пересылает пакет дальше — через провайдера, магистральные каналы и т.д., пока тот не достигнет сети назначения. | - **IPv4**, **IPv6** — основные протоколы адресации<br>- **ICMP** — диагностика (ping, traceroute)<br>- **IPsec** — шифрование на сетевом уровне<br>- **OSPF, BGP, RIP** — протоколы динамической маршрутизации                        |
| Канальный     | L2  | Обеспечивает надёжную передачу кадров между устройствами в одной локальной сети (одном широковещательном домене). Добавляет физическую адресацию (MAC) и контроль ошибок.     | Кадр (frame)         | Компьютер `AA:BB:CC:11:22:33` хочет отправить кадр компьютеру `DD:EE:FF:44:55:66` в той же подсети. Он формирует Ethernet-кадр:                                                                                                                                                                                                      | - Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11)<br>- Протоколы коммутации: STP, LACP<br>- Адресация: MAC-адреса (например, `00:1A:2B:3C:4D:5E`)<br>- Устройства: коммутаторы (switches), мосты (bridges)                                  |
| Физический    | L1  | Передаёт сырые биты (0 и 1) по физической среде. Не понимает структуры данных — только электрические сигналы, световые импульсы или радиоволны.                               | Бит                  | Когда вы отправляете пакет данных, сетевая карта преобразует логические единицы и нули в электрические импульсы, которые бегут по витой паре к коммутатору. На этом этапе нет адресации, нет проверки ошибок — только «есть сигнал / нет сигнала».                                                                                   |                                                                                                                                                                                                                                        |
