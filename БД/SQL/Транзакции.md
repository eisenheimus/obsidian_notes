
#### Уровни изоляции

| Уровень              | Грязное чтение | Неповторяемое чтение | Фантомы | Что происходит                                                       |
| -------------------- | -------------- | -------------------- | ------- | -------------------------------------------------------------------- |
| **READ UNCOMMITTED** | ✅ Да           | ✅ Да                 | ✅ Да    | Читает незафиксированные данные других транзакций                    |
| **READ COMMITTED**   | ❌ Нет          | ✅ Да                 | ✅ Да    | Читает только зафиксированные данные (стандарт в PostgreSQL, Oracle) |
| **REPEATABLE READ**  | ❌ Нет          | ❌ Нет                | ✅ Да    | Строки, которые прочитал, не меняются (стандарт в MySQL)             |
| **SERIALIZABLE**     | ❌ Нет          | ❌ Нет                | ❌ Нет   | Полная изоляция — транзакции выполняются как последовательные        |


## Грязное чтение (Dirty Read)

На уровне изоляции **READ UNCOMMITTED**

_«Грязное чтение — это когда транзакция читает данные, изменённые другой незавершённой транзакцией. Если та откатится, прочитанные данные окажутся ложными. Защищает уровень READ COMMITTED и выше.»_

пример: 

| Время | Транзакция А (читает баланс)                                       | Транзакция Б (переводит деньги)              |
| ----- | ------------------------------------------------------------------ | -------------------------------------------- |
| t1    |                                                                    | BEGIN;                                       |
| t2    |                                                                    | UPDATE account SET balance = 0 WHERE id = 1; |
| t3    | SELECT balance FROM account WHERE id = 1; (ТУТ ПРОБЛЕМА, читает 0) |                                              |
| t4    |                                                                    | ROLLBACK;                                    |
| t5    | ... продолжает работать с balance = 0                              |                                              |
|       |                                                                    |                                              |
Итог: **А прочитала данные, которых никогда не было.**

_«Грязное чтение — это когда транзакция читает данные, изменённые другой незавершённой транзакцией. Если та откатится, прочитанные данные окажутся ложными. Защищает уровень READ COMMITTED и выше.»_

## Неповторяемое чтение (Non-Repeatable Read)

На уровне изоляции **READ UNCOMMITTED**, **READ COMMITTED**

«Неповторяемое чтение — это когда в одной транзакции при повторном чтении той же строки получаем другое значение, потому что другая транзакция успела изменить и закоммитить эту строку. Защищает уровень REPEATABLE READ.»


| Время | Транзакция А (читает дважды)                                                | Транзакция Б (меняет данные)                  |
| ----- | --------------------------------------------------------------------------- | --------------------------------------------- |
| t1    | BEGIN;                                                                      |                                               |
| t2    | SELECT balance FROM account WHERE id = 1;  // результат 100                 |                                               |
| t3    |                                                                             | BEGIN;                                        |
| t4    |                                                                             | UPDATE account SET balance = 50 WHERE id = 1; |
| t5    |                                                                             | COMMIT;                                       |
| t6    | SELECT balance FROM account WHERE id = 1; // Результат 50 (другое значение) |                                               |
| t7    |                                                                             |                                               |
итог: В одной транзакции получили разные результаты.

_«Неповторяемое чтение — это когда в одной транзакции при повторном чтении той же строки получаем другое значение, потому что другая транзакция успела изменить и закоммитить эту строку. Защищает уровень REPEATABLE READ.»_

## Фантомы (Phantom Read)

На уровне изоляции **READ UNCOMMITTED**, **READ COMMITTED**, **REPEATABLE READ**

Одна транзакция дважды выполняет **один и тот же запрос с условием (WHERE)**, а между выполнениями **другая транзакция добавляет или удаляет строки, подходящие под это условие**. Появляются или исчезают "фантомные" строки.


| Время | Транзакция А (считает строки)                                                 | Транзакция Б (добавляет строку)             |
| ----- | ----------------------------------------------------------------------------- | ------------------------------------------- |
| t1    | BEGIN;                                                                        |                                             |
| t2    | SELECT COUNT(*) FROM account WHERE balance > 0; // результат 3                |                                             |
| t3    |                                                                               | BEGIN;                                      |
| t4    |                                                                               | INSERT INTO account (balance) VALUES (100); |
| t5    |                                                                               | COMMIT;                                     |
| t6    | SELECT COUNT(*) FROM account WHERE balance > 0; // результат 4 (Новая строка) |                                             |
| t7    |                                                                               |                                             |
|       |                                                                               |                                             |
|       |                                                                               |                                             |
Итог: В одной транзакции — разное количество строк.

_«Фантомы — это когда при повторном выполнении того же запроса с условием появляются или исчезают строки, потому что другая транзакция вставила или удалила данные, подходящие под условие. Защищает только уровень SERIALIZABLE.»_
