
Фабула
1. Проверить наличие диска lsblk -f
2. Создать раздел - fdisk /dev/sdb
3. Смонтировать диск в каталог к хосту proxmox - mount /dev/sdb1 /mnt/hdd_logs
4. Пробросить через конфиг (/etc/pve/lxc/[id].conf)

**Создание раздела и форматирование диска на хосте(на самом ProxMox)**

```bash
# Просмотреть подключенные устройства и их FS
lsblk -f
```

```Bash
# Создаем раздел (можно нажать Enter на все вопросы для полного диска)

# Краткая шпаргалка по fdisk:
# `n` - new partition (новый раздел)
# `p` - primary partition (основной раздел)
# `l` - список доступных команд
# `d` - удалить раздел
# `w` - записать (save) и выйти
# `q` - выйти без сохранения

fdisk /dev/sdb
```

```bash
# Форматируем в ext4 (надежно для логов)
mkfs.ext4 /dev/sdb1
```

```bash
# Проверям корректно ли создался раздел и его FS
# `-h` — человеческий формат (ГБ, МБ)
# `-T` — показать тип файловой системы
df -hT
```

```bash
# Создайте точку монтирования на хосте
mkdir -p /mnt/hdd_logs
```

```bash
# Монтируем диск в созданную директорию /mnt/hdd_logs
mount /dev/sdb1 /mnt/hdd_logs
```


**Настройте автозагрузку fstab( диск монтировался после перезагрузки сервера)**

```bash 
# Выводи UUID диска
blkid /dev/sdb1 

# Правим конфиг
nano /etc/fstab
```

```ini
# /etc/fstab
UUID=ВАШ-UUID  /mnt/hdd_logs  ext4  defaults  0  2
```

```bash
# Проверка корректности конфига
mount -a
```

```bash
# Проверяем какие FS подмонтированы
mount | grep /dev/sdb
```

```bash 
# Определить тип ФС на устройстве
file -s /dev/sdb1
```

**Монтирование диска в LXC/VM**

```bash
# Останавливаем контейнер
pct stop [id контейнера]
```

```bash
# Открываем конфиг контейнера 
nano /etc/pve/lxc/<CTID>.conf
```

```bash
#Добавьте строку в конец:
mp0: /mnt/hdd_logs,mp=/mnt/logs
```

```bash 
# Запустить контейнер
pct start <CTID>
```

```bash
# Заходим в контейнер
pct enter <CTID>
```

```bash
# Проверям права на папку
# `ls -ld` показывает подробную информацию о самой папке или файле
ls -ld /mnt/logs
```

```bash 
# Если ругается на `Permission denied`, попробуйте выдать права внутри контейнера
chmod 777 /mnt/logs
# Или лучше смените владельца на пользователя, от имени которого работает приложение
chown -R 1000:1000 /mnt/logs
```
