`sed '5d' file.txt`

# добавить строку **после** 2-й
`sed '2a\Новая строка' file.txt` 

# вывести строки с 3 по 7
`sed -n '3,7p' file.txt` 

# **Работа с несколькими командами**
sed -e 's/foo/bar/' -e 's/baz/qux/' file.txt 

# Ключи:
# -i — редактировать файл **на месте** (изменить оригинал)
# -n — подавить автоматический вывод; использовать только с командой `p` для выборочной печати.
# -e — выполнить несколько команд подряд.
# -f script.sed` — выполнить команды из файла.

#Аналзи DNS
# Вывод данных о DNS
dig google.com 

# Вывод данных о DNS только ip
dig goole.com +short

# Вывод данных о серевер обработки почты. Доступны записи [A, AAAA, MX, SOA]
dig google.com MX +short

# A - IPv4-адрес хоста
# AAAA - IPv6-адрес хоста
# MX - Почтовые серверы (Mail eXchanger) с приоритетом.
# CNAME - Каноническое имя (алиас → основное имя).
# NS - Серверы имён (Name Servers), авторитативные для зоны.
# TXT - Текстовые записи: SPF, DKIM, DMARC, верификация домена.
# SOA - Start of Authority — параметры зоны (первичный сервер, администратор, сериализация).
# PTR - Обратная запись (IP → домен), используется в dig -x.
# SRV - Сервисные записи: указывают хост/порт/протокол сервиса.
# CAA - Certificate Authority Authorization — какие CA могут выдавать сертификаты.

# Cигналы kill
Увидеть его можно командой: `kill -l` (выведет номера и имена).

#### Стандартные сигналы (от 1 до 31)

|№|Имя|Действие по умолчанию|Описание|
|---|---|---|---|
|1|SIGHUP|Завершение|Обрыв соединения (например, закрылся терминал). Демоны часто перезагружают конфиги.|
|2|SIGINT|Завершение|Прерывание с клавиатуры (`Ctrl+C`).|
|3|SIGQUIT|Завершение + дамп ядра|Выход с клавиатуры (`Ctrl+\`). Создаёт дамп памяти.|
|4|SIGILL|Завершение + дамп ядра|Недопустимая инструкция процессора.|
|5|SIGTRAP|Завершение + дамп ядра|Точка останова (для отладчиков).|
|6|SIGABRT|Завершение + дамп ядра|Аварийное завершение (вызов `abort()`).|
|7|SIGBUS|Завершение + дамп ядра|Ошибка шины (неправильное выравнивание данных в памяти).|
|8|SIGFPE|Завершение + дамп ядра|Ошибка плавающей точки (деление на ноль).|
|9|**SIGKILL**|Завершение|**Принудительное убийство (не перехватывается!)**|
|10|SIGUSR1|Завершение|Пользовательский сигнал №1 (определяет программист).|
|11|SIGSEGV|Завершение + дамп ядра|Нарушение сегментации (обращение к чужой памяти).|
|12|SIGUSR2|Завершение|Пользовательский сигнал №2.|
|13|SIGPIPE|Завершение|Запись в разорванный канал (например, закрыли чтение из пайпа).|
|14|SIGALRM|Завершение|Таймер сработал (от `alarm()`).|
|15|SIGTERM|Завершение|**Вежливый запрос на завершение** (по умолчанию для `kill` без флагов).|
|16|SIGSTKFLT|Завершение|Ошибка стека сопроцессора (устарело).|
|17|SIGCHLD|Игнорируется|Дочерний процесс завершён или остановлен.|
|18|SIGCONT|Продолжить|Продолжить выполнение после SIGSTOP.|
|19|**SIGSTOP**|Остановка|**Принудительная остановка процесса (не перехватывается!)**|
|20|SIGTSTP|Остановка|Stop с клавиатуры (`Ctrl+Z`).|
|21|SIGTTIN|Остановка|Фоновый процесс читает с терминала.|
|22|SIGTTOU|Остановка|Фоновый процесс пишет в терминал.|
|23|SIGURG|Игнорируется|Срочные данные в сокете.|
|24|SIGXCPU|Завершение + дамп ядра|Превышение лимита CPU (setrlimit).|
|25|SIGXFSZ|Завершение + дамп ядра|Превышение лимита размера файла.|
|26|SIGVTALRM|Завершение|Виртуальный таймер истек.|
|27|SIGPROF|Завершение|Профилирующий таймер истек.|
|28|SIGWINCH|Игнорируется|Изменение размера окна терминала.|
|29|SIGIO|Игнорируется|Событие ввода-вывода (асинхронный ввод/вывод).|
|30|SIGPWR|Завершение|Сбой питания (обычно игнорируется).|
|31|SIGSYS|Завершение + дамп ядра|Недопустимый системный вызов.|

### 1. Управление процессами

- `top` – динамический просмотр активности процессов.
- `htop` – улучшенная версия top с интерфейсом и фильтрацией (устанавливается отдельно).
- `ps aux` – список всех запущенных процессов.
- `kill [PID]` – завершение процесса по ID.
- `kill -9 [PID]` – принудительное завершение.
- `pkill [имя_процесса]` – завершение по имени.
- `pgrep [имя_процесса]` – поиск процессов по имени.

---

### 💾 2. Работа с дисками и файловой системой

- `df -h` – свободное место на дисках.
- `du -sh /path` – размер указанной директории или файла.
- `lsblk` – отображение блочных устройств в виде дерева.
- `smartctl -a /dev/sda` – диагностика состояния диска (SMART).
- `find /path -name "*.log"` – поиск файлов по маске.
- `grep "text" file.txt` – поиск текста в файле.
- `cat`, `less`, `touch`, `cp`, `mv`, `rm`, `mkdir`, `rmdir` – базовые операции с файлами и папками.
- `pwd`, `cd`, `ls` – навигация по файловой системе.

---

### 🧠 3. Мониторинг ресурсов и памяти

- `free -h` – использование оперативной памяти.
- `vmstat` – статистика по памяти, процессам, I/O.
- `iostat` – статистика по дисковому I/O.
- `iotop` – мониторинг дисковой активности в реальном времени.
- `dmesg` – логи ядра (в т.ч. об оборудовании).

---

### 🌐 4. Сетевые команды

- `ip a` – информация о сетевых интерфейсах.
- `ifconfig` – устаревшая альтернатива `ip a`.
- `ping [адрес]` – проверка доступности хоста.
- `traceroute [адрес]` – трассировка маршрута до хоста.
- `mtr [адрес]` – трассировка в реальном времени.
- `netstat -tulnp` – открытые порты и связанные процессы.
- `ss -tulnp` – современная и быстрая замена `netstat`.
- `netstat -rn` – таблица маршрутизации.
- `dig`, `nslookup` – DNS-запросы.
- `curl` – отправка HTTP-запросов:
    
    - `-C -` – возобновление загрузки,
    - `-u user:pass` – аутентификация,
    - `-I` – заголовки ответа,
    - `-H` – кастомные заголовки,
    - `-d` – POST-данные.
    
- `wget` – загрузка файлов.
- `tcpdump -i any port 9100` – перехват сетевого трафика.

---

### 📜 5. Логи и диагностика системы

- `/var/log/auth.log` – попытки входа и привилегии.
- `/var/log/kern.log` – логи ядра.
- `journalctl -xeu [сервис]` – просмотр логов systemd.
- `uname -a` – информация о ядре и ОС.
- `hostname` – имя хоста.
- `uptime` – время непрерывной работы.
- `who`, `w`, `last` – информация о подключённых и недавно заходивших пользователях.

---

### 🔐 6. Управление пользователями и правами

- `sudo` – выполнение команд от имени root.
- `su` – переключение пользователя.
- `useradd`, `passwd` – создание пользователя и смена пароля.
- `groups` – группы пользователя.
- `chmod` – изменение прав доступа.
- `chown` – смена владельца файла/директории.
