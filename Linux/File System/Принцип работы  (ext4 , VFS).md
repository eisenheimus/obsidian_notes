### Файлы

Файлы, которые вы видите в файловом менеджере или через `ls`, — это **жесткие ссылки**.

Каждая такая ссылка — это просто запись в директории, которая содержит:
   - **Название файла**
   - **Номер (адрес) inode** — уникального идентификатора файла на диске.

 **Суть:** Имя файла и его содержимое — это разные вещи. Имя — это просто "табличка", ведущая к inode.

---

### Inode

**Inode** — это структура данных на диске. В ней хранится:
	 -  Счетчик жестких ссылок (`nlink`) — сколько имен ведут на этот inode.
	 
     -  Метаданные:
            - Права доступа (`rwxr-xr-x`)
	       - Владелец и группа (UID / GID)
	       - Размер файла в байтах
	       - Временные метки (создание, модификация, последний доступ)
	       
	 -  Адреса блоков с данными на диске (где реально лежит содержимое файла).
	 

> **Важно:** В inode **не хранится** ни имя файла, ни его содержимое.

> Количество inode ограничено:
 - При форматировании диска создается фиксированное количество inode.
 - Если inode закончатся, создавать новые файлы будет нельзя, даже если свободное место на диске еще есть.
 - Обычное максимальное значение — около **2.3 миллиардов** (зависит от размера файловой системы).


```bash
# Как проверить количество inode -сколько inode всего, сколько использовано и сколько свободно
df -i
```

#### 5. Что происходит при открытии файла (Файловый дескриптор)

Когда процесс открывает файл (через `open()`), ядро создает **файловый дескриптор (FD)**.

- **FD — это просто число** (например, 3, 4, 5...), которое является индексом в таблице открытых файлов процесса.
    
- Через этот индекс процесс находит структуру ядра, которая уже содержит:
    
    - **Ссылку на inode** файла
        
    - **Текущую позицию чтения/записи** (курсор)
        
    - **Режим открытия** (чтение, запись, добавление в конец)
        

> **Разница:** Жесткая ссылка — это постоянная запись на диске. Файловый дескриптор — это временная "ручка" у процесса, пока файл открыт.

#### 6. Диагностика: кто открыл файл и что зависло

- **Посмотреть, какой процесс держит файл:**
    
    bash
    
    lsof /home/file.txt
    
- **Найти "висячие" (удаленные, но открытые) файлы:**
    
    bash
    
    lsof +L1   # Показывает файлы с одной или нулевой ссылкой
    lsof +L0   # Показывает файлы, у которых счетчик ссылок = 0 (удаленные)
    
    _Это файлы-призраки, которые занимают место на диске, но не видны в проводнике._
    

#### 7. Жизненный цикл файла (когда он удаляется)

- Когда вы удаляете файл (`rm`), удаляется **жесткая ссылка** (имя).
    
- Счетчик ссылок в inode уменьшается на 1.
    
- **Только когда счетчик ссылок достигает 0:**
    
    - Данные на диске помечаются как свободные.
        
    - Сам inode освобождается.
        
    - Файл окончательно исчезает с диска.
        

> **Исключение:** Если файл удален (`nlink = 0`), но какой-то процесс все еще держит его открытым (FD висит), данные остаются на диске. Место освободится только когда процесс закроет файл.

### Дополнительные пункты (для углубления), которые вы уже знаете

#### 8. Стандартные файловые дескрипторы (0, 1, 2)

У каждого процесса при запуске уже открыты три изолированных дескриптора:

- **0 (stdin)** — стандартный ввод (обычно клавиатура)
    
- **1 (stdout)** — стандартный вывод (обычно экран)
    
- **2 (stderr)** — вывод ошибок (обычно экран)
    

Они подчиняются тем же правилам: это просто FD, указывающие на inode терминала.

#### 9. VFS (Virtual File System) — почему это всё работает везде

Описанные выше механизмы работают не только для `ext4`, а для **всех** файловых систем в Linux (XFS, btrfs, NTFS, FAT32 и даже `/proc`).

- **VFS** — это прослойка в ядре, которая дает универсальный интерфейс (`open`, `read`, `write`) для любых ФС.
    
- Благодаря VFS в Linux работает принцип **"все есть файл"** (даже процессы и устройства).

### Резюме для памяти

- **Жесткая ссылка** = имя файла (в проводнике).
    
- **Inode** = паспорт файла (метаданные + где лежат данные).
    
- **Файловый дескриптор** = номер пропуска у процесса на открытый файл (хранит позицию курсора).
    
- Файл живет, пока **счетчик ссылок в inode > 0**.
    
- Если inode = 0, а FD есть — файл-призрак (`lsof +L0`).
